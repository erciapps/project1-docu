# PRESENTACIÓN Y ENTREGA DEL PROYECTO

## CONDICIONES DE PRESENTACIÓN Y ENTREGA DEL PROYECTO PARA SUPERAR EL PROYECTO
::::note
1. ES **OBLIGATORIO** DISPONER DEL SERVICIO DE `WEBSERVER`, `NOTIFYSERVER` Y EN EL CASO DE TENER QR, `QRSERVER`. TODOS ELLOS FUNCIONALES.
2. LA APLICACIÓN DEBE REALIZAR TODOS ESTOS PASOS DE FORMA **OBLIGATORIA**: 
- REGISTRAR USUARIO
- ENVIAR EMAIL CON CÓDIGO DE VALIDACIÓN.
- VALIDAR USUARIO CON CÓDIGO
- REALIZAR LOGIN
- REALIZAR LA COMPRAR DE UN TICKET
- ENVIAR EMAIL CON CÓDIGO DE TICKET Y QR (aún que no se use luego el lector de QR)
- VALIDAR CÓDIGO (en mismo programa cliente mediante swing, con página web, o con lector QR desde móvil)
- ENVIAR EMAIL INDICANDO QUE SE HA REALIZADO LA ENTRADA AL EVENTO   
**Si no hace alguno de estos pasos, se debe corregir para la entrega de recuperación**
::::


## PRESENTACIÓN
* La presentación debe consistir en mostrar todo el proceso desde que un usuario se registra, realiza login, compra
al menos un ticket, y se valida ese ticket (los pasos mostrados en el apartado anterior)

* Se dispondrá de un total de **8 MINUTOS MAX.** para mostrar el proceso, junto a **5 MINUTOS MAX.** sobre preguntas sobre el código
de las siguientes partes de la aplicación (evaluable).
